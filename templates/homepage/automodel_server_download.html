1.0 Installing AutoModel Server
----------------------------------

    The AutoModel Server is designed to run only on GNU / Linux environment, before installing it is necessary to install other software that are required for the correct execution of AutoModel Server.

    Os softwares necessários para a execução do AutoModel Server são:
    
        * Python  >= 2.X (http://www.python.org/)
        * RPyC >= 3.x (http://rpyc.readthedocs.org/)
        * Modeller >= 9.7 (salilab.org/modeller/)
        * Procheck (http://www.ebi.ac.uk/thornton-srv/software/PROCHECK/download.html)
        * python-matplotlib (http://matplotlib.org/)
    
    Following are information about the installation of these softwares on Linux.

1.1 Installing Python, RPyC, Modeller e python-matplotlib:
-------------------------------------------------------------
  The best way to install this software is to consult the official documentation of each. Normally Python is already part of the package of the major Linux distributions, you should pay attention, in the case of Python, the AutoModel is only compatible with Python version 2.

    When installing Modeller remember that he possessed a key to be informed at the time of installation.

    Installing python-matplotlib on Ubuntu Linux can be accomplished with the following command:
        
        $ sudo apt-get install python-matplotlib


1.2 Installing PROCHECK
-----------------------------------

   The PROCHECK software is responsible for the stereochemical analysis of models generated by AutoModel. The PROCHECK was developed in the 1990s and has a peculiar installation, comparing the installation of other libraries and programs required for the implementation of AutoModel.

    When installing the PROCHECK on a Debian-based operating system like Ubuntu, the installation of the following dependencies is required:

        * build-essential
        * csh
        * fort77

    On Debian-based operating systems can install PROCHECK as follows:

        - With the terminal navigate to the folder where you downloaded the file procheck.tar.gz.

        - Unzip the contents of procheck.tar.gz file with the command:
          $ tar -xvf procheck.tar.gz
          
        - Enter Procheck folder that was created with the previous command with the command:
          $ cd procheck
          
        - Install the libraries required for the compilation of Procheck
          $ sudo apt-get install build-essential csh fort77

        - Compile PROCHECK:
          $ make
          
        - Move the proshock folder to / usr / share:
          $ cd ..
          $ sudo mv procheck /usr/share
          
        - Create a script for the implementation of PROCHECK compatible with Linux:
          $ sudo touch /usr/bin/procheck
          $ sudo su
          # echo "csh /usr/share/procheck/procheck.scr \$1 \$2" > /usr/bin/procheck
          # chmod +x /usr/bin/procheck
          
        - Create or edit the file ~ / .bashrc and add the following content:
          set    prodir =  /usr/share/procheck
          setenv prodir   '/usr/share/procheck'
          alias procheck      $prodir'/procheck.scr'
          alias procheck_comp $prodir'/procheck_comp.scr'
          alias procheck_nmr  $prodir'/procheck_nmr.scr'
          alias proplot       $prodir'/proplot.scr'
          alias proplot_comp  $prodir'/proplot_comp.scr'
          alias proplot_nmr   $prodir'/proplot_nmr.scr'
          alias aquapro       $prodir'/aquapro.scr'
          alias gfac2pdb      $prodir'/gfac2pdb.scr'
          alias viol2pdb      $prodir'/viol2pdb.scr'
          alias wirplot       $prodir'/wirplot.scr'
      
   Okay, now the PROCHECK is ready for use.

After installing these components, just untar the automodel-server (preferably in home folder) and run the file serverup.sh (2x clicking on or by the terminal with the command sh serverup.sh).
